The Problem

Coordination chaos and a broken sign-up flow

Users struggled to coordinate group dances across different schedules, time zones, and experience levels. But the platform itself made things worse — with unclear navigation, no pricing transparency, and a broken sign-up flow that redirected users to a contact form instead of letting them create an account.

Result: Low trust and high drop-off rates before users even completed onboarding. We hypothesized that the current user flow was not as streamlined as it could be and set out to validate this through user interviews and usability testing on bollystep.com.
Goal

Clear paths for three distinct roles

Simplify onboarding, clarify the platform's value early, create separate experiences for Dancers, Organizers, and Choreographers, and enable users to confidently create and manage dance events.

Research

Three methods to surface the real friction

User Interviews (6 participants)

Conducted to understand user needs and pain points. Revealed that coordination challenges and lack of platform clarity were the key barriers preventing users from moving forward on the platform.

Heuristic Evaluation

Evaluated the platform using established usability heuristics. Found significant issues in navigation consistency, unclear flows, and a lack of trust signals throughout the experience.

Content Audit

Reviewed all content across the landing page and client-facing pages for clarity and structure. Found dense information, unclear messaging, and poor content hierarchy creating cognitive overload for new users.

Affinity Mapping

Organized research data to identify patterns across user behaviors and pain points. Grouped insights revealed four key themes: navigation confusion, onboarding friction, lack of trust, and coordination challenges.

Key Insights

Five critical barriers to user trust

📝

Expected instant sign-up

Users expected to create an account directly — not fill out a contact form that redirected them externally.

🗺️

Navigation felt overwhelming

Too many options with unclear hierarchy left users unsure of where to go or what the platform was for.

👤

Roles were unclear

Users couldn't tell whether the platform was for them — Dancer, Organizer, or Choreographer — from the homepage.

💰

No pricing transparency

Hidden pricing and no previews reduced trust. Users weren't willing to commit without knowing what they were signing up for.

📅

Coordination was the core pain point

Managing group dances across different schedules, time zones, and skill levels was the fundamental challenge the product needed to solve — but the existing platform barely addressed it.

Site Architecture

Restructured for three distinct user roles

The original navigation was a catch-all — one muddled structure trying to serve every user type. The redesigned architecture creates clear, separate entry points for each role:

  • Introduced dedicated pages for Organizers, Dancers, and Choreographers with tailored content and flows
  • Added a dedicated CTA: "Apply to be a Choreographer" to support onboarding of new talent
  • Replaced the vague "Join Now" button with a clear Log In / Sign Up flow accessible directly from the homepage

User Flows

Reducing friction from discovery to event creation

The original flow asked too much of users before giving them any value. Key problems and how they were solved:

  • Users expected instant account creation but were redirected to a contact form — fixed with direct sign-up access
  • Additional steps in the app increased effort and caused frustration — reduced unnecessary steps
  • Organizers needed a faster, low-effort flow during time-sensitive event planning — combined the marketing site and platform into a single seamless flow

Final Design

Three focused design decisions

Role-Based Navigation

To reduce confusion and help users quickly understand how the platform applies to them. Created clear entry points for Organizers, Dancers, and Choreographers with tailored content and flows — so the first visit immediately answers "is this for me?"

[Image Placeholder — Role-Based Navigation Screens]

Explore Before You Commit

To build trust and reduce hesitation before event creation. Users can browse dances and preview content before committing — addressing the #1 trust barrier identified in research.

[Image Placeholder — Explore Dance Feature Screens]

Simplified Onboarding

To reduce friction and speed up the user journey. Combined the marketing and client-facing platform into a single seamless flow and removed the NDA barrier that was blocking users from signing up.

Usability Testing

Two rounds — existing site vs. redesign

Conducted two rounds of usability testing: Round 1 with the existing website, Round 2 with the proposed mid-fi prototype. Observed significant gains in clarity, navigation understanding, and task success rates. Feedback from R2 was then implemented in the high-fidelity design.

[Image Placeholder — Usability Testing Notes / Screens]

Key findings from testing:

  • Users understood their roles faster with the new navigation structure
  • Reduced confusion about where to go to sign up or log in
  • Improved onboarding success — more users completed the flow without assistance
  • Increased engagement with the "Explore" content before event creation
Outcome

Reduced friction, improved trust

The redesign reduced onboarding friction and improved usability across key flows. Provided clear pathways for dancers, choreographers, and organizers — strengthening trust through clearer navigation, transparency, and better user adoption.

Reflection

Simplifying flows is about building trust, not just usability

I learned that simplifying user flows isn't just about usability — it's about building trust early. Aligning navigation, onboarding, and content with user expectations created a more intuitive and confident experience.

If I had more time, I would have run more rounds of usability testing on the high-fidelity prototype, explored A/B testing for the role-based entry points, and dug deeper into the coordination features that remain the platform's core value proposition.
